# Career package — Technical Intelligence Analyst (SIA, OpenAI)

This folder is a complete, end-to-end kit for landing and then *doing* the
**Technical Intelligence Analyst** role on OpenAI's **Strategic Intelligence &
Analysis (SIA)** team. It's organized by the three phases of the journey.

> Everything about the **flagship project** (numbers, detectors, queries) is factual and
> verifiable in this repo. Anything in `[brackets]` in the application docs is a
> **placeholder** for your personal specifics — fill those in before sending.

## 1 · Apply
- [`application/cover-letter.md`](application/cover-letter.md) — full letter + short form-field version + referral/email blurb (primary SIA TIA role).
- [`application/cover-letter-variants.md`](application/cover-letter-variants.md) — tailored letters for **adjacent OpenAI roles** (Technical Abuse Investigator, Technical Threat Investigator, Quantitative Intelligence Analyst, AI Emerging Risks Analyst) + a role chooser.
- [`application/resume-and-linkedin.md`](application/resume-and-linkedin.md) — résumé bullets, summary line, ATS skills, LinkedIn headline + About, and "why OpenAI / why this role" answers.
- [`llm-log-triage-case-study.pdf`](llm-log-triage-case-study.pdf) — one-page PDF to attach.

## 2 · Interview
- [`interview-prep.md`](interview-prep.md) — the dossier: repo→JD mapping, framework cheat-sheets, 18 Q&As (incl. SIA-specific), STAR stories, leadership talking points, study plan.
- [`interview/mock-interview.md`](interview/mock-interview.md) — a **full mock loop** (recruiter screen → technical screen → SQL/Python exercise → behavioral) with model answers, follow-up probes, and what each interviewer is listening for.
- [`interview/demo-script.md`](interview/demo-script.md) — a 6-minute live/Loom walkthrough of the repo, beat-by-beat, with exact commands.
- [`interview/demo-narration.md`](interview/demo-narration.md) — a **record-ready teleprompter** version of the demo with cumulative timing cues.
- [`interview/cheat-sheet.md`](interview/cheat-sheet.md) — one-page memorize-this (frameworks, numbers, definitions, 30-second pitch). Printable [`cheat-sheet.pdf`](interview/cheat-sheet.pdf) alongside it.

## 3 · Do the job (day one onward)
- [`playbook/analyst-runbook.md`](playbook/analyst-runbook.md) — the operational SOP: daily/weekly workflow, per-OWASP investigation playbooks, severity rubric, escalation, intel-brief + finding templates, and when to build a detector vs an anomaly model.
- [`playbook/onboarding-30-60-90.md`](playbook/onboarding-30-60-90.md) — a concrete first-90-days plan.

## The through-line
The same skills appear in all three phases: **ingest messy data → detect adversarial
patterns mapped to OWASP/MITRE ATLAS → prioritize by severity → communicate as
decision-ready intelligence.** The flagship proves you can do it; the dossier proves you
can talk about it; the playbook proves you know what the job actually looks like.
